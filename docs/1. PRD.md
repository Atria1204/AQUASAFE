# Product Requirement Document (PRD) - AquaSafe

## 1. Product Overview & Vision
AquaSafe is an IoT-based aquaculture monitoring and automation system designed to help pond owners (such as gurame or tilapia farmers) monitor water conditions in real-time, view live video feeds, and automate the feeding process. 

The goal of the system is to minimize fish mortality, optimize feed conversion ratios, and reduce manual labor by providing remote control and intelligent telemetry monitoring.

---

## 2. Target Users & Personas
* **Pond Owner / Farmer:** Needs a clear, glanceable dashboard to monitor multiple ponds from anywhere, check if parameters are safe, and manually trigger feed when necessary.
* **Aquacultural Technician:** Needs access to detailed historical logs (temperature, pH, TDS) to diagnose water quality trends and calibrate sensor offsets.

---

## 3. Core Features & Scope

### 3.1. Real-Time Telemetry Monitoring
* **Water Parameters:** Suhu (Temperature), pH, DO (Dissolved Oxygen), TDS (Total Dissolved Solids), and water flow rates (flow1, flow2, flow3, flow4).
* **Safe Ranges & Alerts:** Visually warn the user when parameters fall outside safe thresholds.
* **WebSocket Updates:** Water metrics must update live without requiring page refreshes.

### 3.2. Automated Feeding Control
* **Scheduled Feedings:** Allow users to set specific hours for morning and afternoon feedings.
* **Manual Override:** Provide a "Beri Pakan Sekarang" button to trigger instant feeding with a custom amount of feed in grams.
* **Stok Notification:** Alert the user if the physical hopper runs out of feed (monitored by an infrared sensor on the ESP32 feeder).

### 3.3. CCTV Video Streaming
* **ESP32-CAM Integration:** Support live video streaming directly from the ESP32-cam module associated with the selected pond.
* **Camera Control:** Toggle camera power remotely to save energy when not viewing.

### 3.4. Device & Calibration Settings
* **Multi-device Support:** Let users add and select different ponds/devices.
* **Calibration Offsets:** Allow manual sensor adjustment parameters (offset_temp, offset_ph, offset_tds) to correct sensor drift without rewriting firmware.

---

## 4. Technical Constraints & Out of Scope
* **Hardware Offline State:** The system should gracefully handle cases where the ESP32 client is offline or loses Wi-Fi connection.
* **Mobile Support:** Responsive Web Application (optimized for both desktop and smartphones).
* **No Multi-farm Hierarchy:** (Out of Scope for V1) Supporting multiple separate physical farms under a single enterprise is not required; it is currently 1-to-1 between users and devices.
